# Briefing — Your First Week at Northwind Engineering

*One-page "start here" briefing generated from the onboarding notebook (Studio → Briefing doc), then tidied. Every point is answerable in the notebook with a citation.*

## Day 1 — get set up
- Install the standard toolchain and **turn on MFA** for Google and VPN before touching internal systems `[S1][S8]`.
- Encrypt your disk and set up a password manager — required by the security basics policy `[S8]`.
- Join your team's Slack channels from the directory `[S7]`.

## First week — get access
- Request production data access through the IT portal with a business justification; your lead **then** Security approves it, and it expires after 90 days `[S2]`.
- Access is least-privilege by default — ask only for what your first tasks need `[S2]`.

## How we work
- **Never push to `main`.** Branch → pull request → at least one approving review → merge `[S6]`.
- Production deploys follow the deploy runbook and only run after CI is green `[S3]`.

## On-call & escalation
- On-call rotates weekly, handed over Mondays. Escalate primary → secondary → engineering manager `[S4]`.

## Time off
- 20 days PTO/year; request 3+ day leave two weeks ahead via the HR portal; up to 5 days carry over `[S5]`.
