Decision Knowledge Base (Vendor / Plan / Framework)

Week 3 M8 · Research with NotebookLM Solo Worked sample
Problem statement
You're weighing a big decision (a vendor, a health plan, a framework) and the material is scattered across PDFs and pages. Build a NotebookLM knowledge base over those sources and ask the trade-off questions that matter; every answer cites where it came from. Success: a notebook + a one-page cited decision brief + an Audio Overview.
What you build
A research knowledge base that pulls all the decision material into one place. For this reference we're choosing a customer-support (helpdesk) vendor between three options — but the same shape works for a health plan, a cloud provider, or a frontend framework. You load each vendor's pricing PDF, feature page, docs and a third-party review, then ask the trade-off questions that actually decide it: “which is cheapest at 15 agents?”, “which one locks me in?” — and every answer cites the exact page. Real RAG, so your decision rests on sourced facts, not a sales rep's memory. Worked reference: choosing between HelpHub, DeskFlow, and CaseNest.
What ships in this folder
sources.md

The exact list of sources that went into the notebook.

 qa.md

Real questions with the cited answers NotebookLM gave.

 briefing.md

A one-page briefing / study guide you could hand over.
Build it in NotebookLM
No API key, no code — NotebookLM is free with a Google account.

Go to notebooklm.google.com, sign in, Create new notebook.

Add source for each option: upload the pricing PDF, paste the feature/comparison page URL, add a docs page, and paste a third-party review. Cover each option evenly.

Ask the trade-off questions, e.g. “Which option is cheapest for 15 agents, and what's excluded from that price? Cite each.”

Confirm each answer's citation points at the right pricing/feature page. Save into qa.md.

In Studio, generate a Briefing doc into briefing.md and sharpen it into a recommendation; generate an Audio Overview to gut-check the trade-offs.

Where Claude.ai fits
  • Ask Claude to help you list the trade-off questions that actually decide this (cost, lock-in, compliance, support).
  • Paste the cited answers into Claude to turn them into a decision brief aimed at the person who signs off.
How this differs from HO4

HO4 answered from one pasted document. Here NotebookLM retrieves across all the vendor PDFs and pages at once and cites the source of each trade-off claim — the retrieval + citation is what lets you defend the decision.
